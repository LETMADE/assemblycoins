Quickstart

This guide is going to get you up and running with Assembly Coins with the following steps:

  1. Issue a new coin
  2. Transfer coins to your wallet
  3. Check your coin balance

1. Issue a new coin

Specify some details about your new Asset class. All of this metadata will be encoded on the Bitcoin Blockchain, with the exception of your email address. Your email is used to alert you when the coin is ready for transfer.

{% highlight bash %} $ curl https://coins.assembly.com/v1/colors/prepare \ -X POST \ -H "Content-Type: application/json" \ -d '{ "issued_amount": 10000000, "description": "Karefully Krafted Koin for Kats", "coin_name": "KatKoins", "email": "kris@example.com" }' {% endhighlight %}

This will return a response like looks like:

{% highlight javascript %} { "minting_fee": "0.00054808", "issuing_public_address": "1MRuSdYzMHmifxw7FwpYpzcoPDRm2J5VLv", "issuing_private_key": "5KkEKXmPhyzxcgkUqtqdk7id2ijbaXBwmkZ6rCqrQJx5nyWWdLa", "name": "KatKoins" } {% endhighlight %}

You now need to send the minting_fee amount to the issuing_public_address. For the example above, you would need to send 0.0004 BTC to 1EmnqhfvjcAdA71gs2exugXkgHrJw9QcuA.

This small amount of Bitcoin must be sent to the issuing address to pay for miner fees and processing. After your Bitcoin has arrived, a series of transactions will be written for you. These transactions will:

  • Write a multipart statement on the Bitcoin Blockchain declaring your intent.
  • Write an Open Assets issuing transaction to back to the issuing address. You can transfer existing coins later.

Your new coin now lives on the blockchain!

2. Transfer coins to your wallet

You still have full control over the issuing address because you are given a private key in step 1, however, for added security you may want to transfer the coins to a wallet that you control. This transactions will be signed serverside.

{% highlight bash %} $ curl https://coins.assembly.com/v1/transactions/transfer -H "Content-Type: application/json" -X POST -d '{ "to_public_address": "173CJ9wxuZFbJyDbkJ89AfpAkqx5PatxMk", "transfer_amount": 44, "from_private_key": "PRIVATE KEY HERE", "fee_each": 5e-05, "from_public_address": "1MRuSdYzMHmifxw7FwpYpzcoPDRm2J5VLv", "source_address": "1MRuSdYzMHmifxw7FwpYpzcoPDRm2J5VLv" }' {% endhighlight %}

The from_public_address & from_private_key are the address and private key of the issuing address from step 1. For this initial transfer, the source_address is also the from_public_address.

This command queues the transaction to be processed. It can take several minutes to be fully recognized by the blockchain. The command will return immediately but if you specify the callback_url parameter, you will receive a JSON POST with more specific transaction hash information once the transaction has been processed.

{% highlight javascript %} {"result": "Queued"} {% endhighlight %}

Callback Response

{% highlight javascript %} { "transaction_hash":"18812f3e2eebb3a267afd54b8b8c54a8a2a7afe719ec4c5c19342c42793bb52a" } {% endhighlight %}

Once that transaction has processed you'll be the proud owner of 10 Kat Koins!

3. Check your coin balance

Now for the easy part. Lets inspect your wallet and see what coins it contains:

{% highlight bash %} $ curl https://coins.assembly.com/v1/addresses/1BHjGfCcEY1uJMJsHRJDXBdSeo5SbqcHcz {% endhighlight %} Your response: {% highlight javascript %} { "public_address": "1BHjGfCcEY1uJMJsHRJDXBdSeo5SbqcHcz", "assets": [ { "quantity": 10, "color_address": "3N5CbL5qHbu1P2oBEaVsFJ4R1n3UxsV1o3" } ] } {% endhighlight %}

Now you're ready to start diving into the full API documentation and building your own apps using Assembly Coins!
